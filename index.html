Key Gap: The "Manual" Step

Right now, the biggest bottleneck is the manual step between AutoResearch and auto_trader. When autoresearch finds a better config (e.g., OOS Sharpe 2.03), someone must manually extract the parameters and update auto_trader.py. 牧之林 has a Deploy Gate that automates this: if a strategy passes all 5 research gates, it automatically promotes to paper trading.

Implementation Roadmap

Based on impact/effort analysis. Focus on connecting existing code before building new.

  • 1
    Connect Research Gate to AutoResearch
    Impact: HIGH | MegaTrade gate.py already has 5-layer logic. Wire it so autoresearch results automatically pass through cost_x2 + robust + stress + ml_oos gates. This is the single highest-value integration.
  • 2
    Connect Deploy Gate to auto_trader.py
    Impact: HIGH | deploy_gate.py has 5 validation checks. If a config passes Research Gate + Deploy Gate, auto-update auto_trader.py parameters. Eliminates the "Manual" bottleneck.
  • 3
    Review & Activate Runtime Gate
    Impact: MEDIUM | HALT logic was added but also changed trading params (v7.1→v8.14 conflict). Need selective revert, then activate the 4-condition monitoring.
  • 4
    Event-Driven Risk Control
    Impact: MEDIUM | Monitor VIX, oil, macro indicators. Auto-suggest hedging when thresholds breach. 牧之林's 3-button Discord UI (Execute/Pause/Cancel) is a good reference.
  • 5
    Live Dashboard
    Impact: LOW-MED | Real-time view of positions, PnL, system health, recent experiments. MegaTrade dashboard.py exists. Could also build as web dashboard (Dash/Streamlit).
  • 6
    Position Reconciliation
    Impact: LOW | Compare local state vs Alpaca positions. Detect drift. Important for production but less urgent for paper trading phase.

Core Principle

Connect before build. We have 14 components as code that aren't used. The highest ROI is wiring them into the actual pipeline, not building new things. Priority #1 and #2 alone would close the biggest gap between our system and 牧之林.
